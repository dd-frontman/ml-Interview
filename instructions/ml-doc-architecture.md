## Каркас раздела

### 1) Big Picture

- Кратко объяснить роль supervised learning в продуктовой разработке.
- Показать место в общем ML lifecycle: данные -> обучение -> inference -> мониторинг.

### 2) Концепция простым языком

- Объяснение через аналогию с frontend:
  - `y` как ожидаемый UI state;
  - признаки как пропсы/сигналы из backend и клиентской телеметрии.

### 3) Инженерная интерпретация

- Формализовать data/training/inference flow.
- Пояснить, где и почему возникают leakage, overfitting, threshold-проблемы.
- Показать минимально необходимый набор метрик.

### 4) Минимальный пример кода

- Компактный воспроизводимый пример на `scikit-learn`.
- Цель: показать skeleton pipeline за 30-60 секунд чтения.

### 5) Практический пример

- Реалистичный кейс на табличных данных с mixed feature types.
- Использовать `ColumnTransformer + Pipeline`.
- Добавить оценку качества и базовую бизнес-интерпретацию результата.

### 6) Типичные ошибки

- Формат: список "ошибка -> почему это проблема -> как исправить".
- Минимум 5 ошибок, включая leakage и неправильный выбор метрики.

### 7) Cheat-sheet таблица

- Поля: этап, что делаем, инструменты, артефакт, контрольный вопрос.
- Сфокусировать на собеседовании и повседневной практике.

### 8) Official docs

- Встроенный блок `<OfficialDocsLinks />`.
- Ссылки только на официальные первоисточники библиотек/фреймворков.

## Границы и анти-дублирование

- Не дублировать deep-learning детали и MLOps deployment-паттерны.
- Ссылаться на:
  - `mlops/feature-engineering-i-validatsiya` для расширенной валидации;
  - `ml/obuchenie-bez-uchitelya` для unsupervised-сценариев;
  - math-раздел для статистической базы.

## Definition of Ready для публикации

- Frontmatter валиден.
- Все внутренние ссылки проходят `validate-links`.
- Пример кода воспроизводим локально на Python + scikit-learn.
- Документ читается как инженерный гайд, а не академический конспект.

## Big Picture

Python - это универсальный язык программирования с простым синтаксисом и большой экосистемой библиотек.

Для ML Python стал де-факто стандартом, потому что:

- есть mature-стек для данных и моделей (`NumPy`, `Pandas`, `scikit-learn`, `PyTorch`, `TensorFlow`);
- быстрый старт для прототипов;
- удобная интеграция с backend, data pipelines и MLOps-инструментами.

## Краткая история

- 1989: Гвидо ван Россум начал разработку Python.
- Февраль 1991: опубликована версия Python `0.9.0`.
- Октябрь 2000: вышел Python `2.0`.
- Декабрь 2008: вышел Python `3.0` (несовместимость с частью Python 2 кода).
- 1 января 2020: официально завершена поддержка Python 2.

Сейчас в индустрии стандартом считается Python 3.

## Где используется Python

- Backend и API-сервисы (FastAPI, Django, Flask).
- Data engineering и аналитика.
- Машинное обучение и deep learning.
- Автоматизация, скриптинг, DevOps-задачи.
- Тестирование и внутренние инструменты команд.

## Роадмап Python для middle-разработчика

Ниже roadmap по самому языку и инженерной практике на Python.

### Этап 1. Python Core (1-2 недели)

- Типы данных, изменяемость, ссылки на объекты.
- Управляющие конструкции, comprehensions, функции, `*args/**kwargs`.
- Области видимости, исключения, контекстные менеджеры.

Результат:

- уверенно пишешь читаемый Python-код без "магии";
- понимаешь, почему код ведет себя именно так.

### Этап 2. Структуры и стандартная библиотека (1-2 недели)

- `list`, `tuple`, `dict`, `set`, `collections`.
- `pathlib`, `datetime`, `json`, `csv`, `itertools`, `functools`.
- Работа с файлами и потоками данных.

Результат:

- решаешь большинство прикладных задач без сторонних пакетов.

### Этап 3. ООП, архитектура, typing (2-3 недели)

- Классы, dataclass, наследование и композиция.
- Абстракции через `ABC` и `Protocol`.
- Type hints, `mypy`, дизайн API модулей.

Результат:

- пишешь поддерживаемый код и контракты между модулями.

### Этап 4. Инструменты разработчика (1 неделя)

- `venv`, `pip`, зависимости, `pyproject.toml`.
- Линтинг и форматирование (`ruff`, `black`), pre-commit.
- Структура проекта и базовый CI.

Результат:

- воспроизводимая сборка и единый стиль кода в команде.

### Этап 5. Тестирование (1-2 недели)

- `pytest`, фикстуры, параметризация тестов.
- Mock/stub, тестирование ошибок и edge-cases.
- Покрытие и качество тестов (а не только процент).

Результат:

- стабильный refactoring без страха поломок.

### Этап 6. Backend и API-практика (2-3 недели)

- HTTP, REST, валидация схем, сериализация.
- FastAPI/Django на базовом production-уровне.
- Логирование, обработка ошибок, конфиги и env vars.

Результат:

- можешь собрать и поддерживать API-сервис на Python.

### Этап 7. Concurrency и производительность (2 недели)

- `asyncio`, event loop, когда нужен `async/await`.
- Потоки vs процессы, CPU-bound vs I/O-bound задачи.
- Профилирование (`cProfile`), поиск bottleneck.

Результат:

- осознанно выбираешь модель конкурентности и оптимизации.

### Этап 8. Production-ready навыки (1-2 недели)

- Docker-образ, healthchecks, graceful shutdown.
- Метрики и observability.
- Релизы, rollback, troubleshooting инцидентов.

Результат:

- код готов к эксплуатации, а не только к локальному запуску.

## Чеклист middle Python-разработчика

| Область | Что должен уметь middle |
| --- | --- |
| Язык | Писать читаемый код, понимать mutability, исключения, контексты |
| Архитектура | Декомпозировать на модули, проектировать API, применять typing |
| Качество | Писать unit/integration тесты, настраивать lint/format/type-check |
| Backend | Делать API-сервисы, валидировать вход/выход, логировать ошибки |
| Производительность | Выбирать `sync/async`, находить bottleneck, делать базовое профилирование |
| Продакшен | Упаковывать сервис, настраивать CI/CD, мониторить и чинить инциденты |

## План по времени

| Период | Фокус | Ожидаемый итог |
| --- | --- | --- |
| Недели 1-2 | Core + stdlib | Уверенное владение языком |

| Недели 6-8 | Тесты + backend + async | Готовность к реальной командной разработке |

## Что входит в группу

В этом разделе: `5` изменяемых типов и групп типов.

- `list`
- `dict`
- `set`
- `bytearray`
- пользовательские объекты классов

Изменяемый тип значит: объект можно менять без создания нового.

## List

`[1, 2, 3]` Аналогия с JS: обычный массив `[]`

- Упорядоченная изменяемая коллекция
- Можно `append`, `remove`, сортировать, менять элементы по индексу
- Поддерживает срезы: `items[1:3]`
  Срез - это способ взять часть списка по диапазону индексов.
  `items[1:3]` значит: взять элементы с индекса `1` до `3`, не включая `3`.
  Пример: `[10, 20, 30, 40][1:3] -> [20, 30]`
- Может хранить значения разных типов, но в прикладном коде лучше держать один тип элементов
- Поиск элемента через `in` работает линейно, поэтому для частых membership-checks `set` обычно лучше
  ```python
  values_list = [1, 2, 3, 4, 5]
  values_set = {1, 2, 3, 4, 5}

  5 in values_list  # True
  5 in values_set   # True
  ```
- Копия через `items[:]` или `items.copy()` поверхностная, вложенные объекты останутся общими
- Частая ошибка после JS: `list` не имеет методов `map` и `filter` как у массива, вместо этого используют list comprehension
- `list` хорошо подходит для последовательностей, очередей на простом уровне, batch-данных

## Dict

`{"id": 42, "role": "admin"}` Аналогия с JS: обычный объект

- Хранит пары ключ-значение
- Удобен для JSON-подобных структур, конфигов, DTO и payload
- Доступ по ключу быстрый: `user["id"]`
- Ключи должны быть хешируемыми: строка, число, `tuple` без mutable-объектов внутри подходят, `list` не подходит
- При обращении к отсутствующему ключу через `dict[key]` будет `KeyError`, для безопасного чтения часто используют `get`
- Начиная с современных версий Python сохраняет порядок добавления ключей
- Частая ошибка: путать `dict.get("x")` и `dict["x"]`; первое вернет `None` или default, второе может упасть с ошибкой
- Для объединения словарей можно использовать `update()` или оператор `|`

## Set

`{1, 2, 3}` Аналогия с JS: `new Set([1, 2, 3])`

- Хранит только уникальные значения
- Удобен для дедупликации, пересечений и быстрых membership-checks
- Не гарантирует удобный "человеческий" порядок элементов, поэтому для стабильного вывода часто нужен `sorted(...)`
- Хорошо подходит для операций множеств: пересечение `&`, объединение `|`, разность `-`
- Элементы тоже должны быть хешируемыми, поэтому `set([ [1, 2] ])` не сработает
- Частая ошибка: ожидать индексный доступ как у `list`; у `set` нет `items[0]`
- Практический кейс: быстро убрать дубликаты `list(set(values))`, но порядок при этом потеряется

## Bytearray

`bytearray(b"hello")` Аналогия с JS: изменяемый бинарный буфер, близкий по смыслу к работе с `Uint8Array`

- Изменяемая версия `bytes`
- Удобна, когда нужно менять бинарные данные по месту

## Пользовательские объекты классов

```python
class User:
    def __init__(self, name: str) -> None:
        self.name = name
```

Аналогия с JS: это как экземпляр класса `new User()`

- Обычно объекты классов изменяемые: `user.name = "Bob"`
- Это зависит от реализации класса

## Практический пример

```python
user = {"id": 1, "roles": ["admin"]}
user["roles"].append("editor")

print(user)  # {'id': 1, 'roles': ['admin', 'editor']}
```

Здесь меняется исходный объект, а не создается новый.

## Типичные ошибки

1. Думать, что копия `list.copy()` или `dict.copy()` полностью независимая.
   Решение: помнить, что это только поверхностная копия.

2. Ожидать индексный доступ у `set`.
   Решение: `set` нужен для уникальности и membership, а не для порядка.

3. Использовать `dict[key]` там, где ключ может отсутствовать.
   Решение: использовать `get`, если отсутствие ключа - нормальный сценарий.

4. Смешивать в `list` данные разного смысла и типа.
   Решение: для структуры данных лучше `dict`, `tuple`, `dataclass`.

## Cheat-sheet

| Тип | Пример | Особенность |
| --- | --- | --- |
| `list` | `[1, 2, 3]` | Упорядоченная изменяемая коллекция |
| `dict` | `{"id": 1}` | Ключ-значение |
| `set` | `{1, 2, 3}` | Только уникальные элементы |
| `bytearray` | `bytearray(b"abc")` | Изменяемые бинарные данные |
| объект класса | `User("Alice")` | Изменяемость зависит от реализации |
